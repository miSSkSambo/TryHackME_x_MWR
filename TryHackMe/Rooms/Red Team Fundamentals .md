## 📌 Overview

This room introduces the fundamentals of **Red Teaming**, a cybersecurity practice focused on simulating real-world attacks to test an organisation’s ability to detect and respond to threats.

Unlike vulnerability assessments and penetration testing, red teaming focuses on:
- Real attacker behavior
- Stealth and evasion
- Detection and response capabilities of defenders (Blue Team)

---

## 🎯 Learning Objectives

- Understand what red team engagements are
- Identify key stakeholders (Red, Blue, White teams)
- Learn differences between vulnerability assessments, pentesting, and red teaming
- Understand attacker methodologies (TTPs)
- Learn about the Cyber Kill Chain

## 🚀 Summary

Red Teaming provides a **realistic simulation of cyber attacks** by mimicking real-world adversaries. Unlike traditional testing, it focuses on:
- Staying undetected
- Exploiting human, physical, and technical weaknesses
- Testing how well defenders respond to threats

It is a critical practice for improving an organisation’s **overall security posture and incident response capabilities**.
